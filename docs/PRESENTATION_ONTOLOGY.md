# CCOP (범죄 사이버 인텔리전스 시스템) 온톨로지 개념 및 설계 구조 (외부 발표용)

## 📌 1. 도입: 왜 '지식 그래프(Ontology)'가 필요한가?
*   **기존 수사의 한계:** 사이버 범죄(보이스피싱, 사이버도박, 랜섬웨어 등)는 대포통장, 대포폰, 해외 IP를 넘나드는 **연쇄적이고 복잡한 네트워크**를 형성합니다. 기존의 RDB(엑셀 표 형식)로는 수백 개의 통장이 얽힌 10단계 자금 세탁 경로를 찾는데 며칠이 소요됩니다.
*   **CCOP 온톨로지의 해결책:** 세상의 모든 단서(계좌, 전화번호, 사람, 사건)를 **'노드(점)'와 '엣지(선)'의 관계망**으로 정의하여, 흩어진 퍼즐 조각들을 하나의 거대한 범죄 연결망(지식 그래프)으로 입체화합니다. 단 몇 조각의 단서만으로도 숨겨진 공범과 은닉 자금의 경로를 수 초 내에 시각적으로 찾아낼 수 있습니다.

## 🏗 2. CCOP 온톨로지 핵심 아키텍처: V2 4-Layer 모델
CCOP 온톨로지는 수사관의 사고방식(Who, What, How, Evidence)을 그대로 본따 **4개의 계층(Layer)**으로 철저히 분리 설계되었습니다.

### Layer 1: 사건 (Case Layer) ─ *"무슨 일이 일어났는가?"*
*   **개념:** 경찰/검찰 등에 접수된 최상위의 범죄 발생 기록 단위입니다.
*   **주요 엔티티:** `vt_case` (사건번호, 접수일, 죄종: 보이스피싱 결합 사건 등)
*   **역할:** 지식 그래프의 '지도(Map)' 역할을 하며, 모든 피의자와 피해자, 증거를 묶어내는 최상위 바운더리입니다.

### Layer 2: 행위자 (Actor Layer) ─ *"누가 연루되었는가?"*
*   **개념:** 범죄 시스템을 구동하는 물리적, 논리적 실체들입니다.
*   **주요 엔티티:** 
    *   `vt_psn` (인물 - 피의자, 피해자)
    *   `vt_org` (조직 - 범죄 단체, 은행, 통신사)
    *   `vt_persona` (가상 자아 - 텔레그램 익명 계정, 다크웹 ID)
*   **역할:** 사건의 주체이며, 아래 Layer 4(증거/자산)를 소유하거나 움직이는 통제자입니다.

### Layer 3: 행위 및 이벤트 (Action / Event Layer) ─ *"어떤 시계열적 행동을 했는가?"*
*   **개념:** 행위자와 자산 사이에 발생한 **'시간(Time)'이 기록된 트랜잭션**입니다.
*   **주요 엔티티:** 
    *   `vt_transfer` (자금 이체 이벤트 - 송금액, 일시)
    *   `vt_call`, `vt_msg` (통화/메시지 수발신 이벤트 - 지속시간, 일시)
    *   `vt_access` (IP 접속 이벤트)
*   **특징:** 이벤트를 독립된 "노드"로 빼냄으로써, 하나의 이체 내역에 IP, 위치, ATM 단말기 등 수많은 곁가지 단서를 다차원적으로 붙일 수 있습니다. (확장성 극대화)

### Layer 4: 증거 및 자원 (Evidence / Resource Layer) ─ *"어떤 도구와 경로가 쓰였는가?"*
*   **개념:** 범죄에 동원되거나 타겟이 된 **불변의 객관적 증거**물입니다.
*   **주요 엔티티:** 
    *   `vt_bacnt` (금융 계좌), `vt_telno` (전화번호)
    *   `vt_ip` (아이피), `vt_atm` (CCTV가 있는 물리적 인출 위치)
*   **역할:** 실제 수사의 꼬리를 무는(Tracing) 핵심 매개체입니다.

## 🔄 3. 하이브리드 어프로치: 왜 LPG + Semantic Rule 인가?

글로벌 수사 플랫폼(i2, Palantir)의 트렌드에 맞춰, **속도(LPG)**와 **논리(Semantic OWL)**의 완벽한 융합 아키텍처를 채택했습니다.

1.  **시각화 및 자금 추적 속도 = 속성 그래프 (LPG) 사용**
    *   W3C 표준인 RDF(Triple) 모델은 "이체 금액", "날짜" 같은 속성을 표현하기 어려워 직관성이 떨어지고 쿼리 연산(N-hop)이 매우 느립니다.
    *   CCOP는 최하단 DB를 **AgensGraph(LPG)**로 채택하여, **"누가 ➡️ 이체(금액/시간) ➡️ 누구에게"** 라는 물리적 화살표 형태를 100% 직관적으로 유지하여 **미세초(ms) 단위의 궤적 추적**을 보장합니다.
2.  **보이지 않는 룰 설계 = 시맨틱 추론 엔진 (Semantic Reasoning) 적용**
    *   단순히 선을 그리는 것을 넘어, **AI 엔진이 사람을 대신해 숨겨진 공범을 유추**합니다.
    *   (예: 추이성 규칙) A계좌 $\rightarrow$ B계좌 $\rightarrow$ C계좌 로 입출금이 즉시 일어났다면? ➡️ 엔진이 B는 단순 '경유지'로 판단하고, A와 C 사이에 **[자금세탁 종착지]** 라는 새로운 가상 추론 엣지(점선)를 시스템이 자동으로 그려 수사관에게 브리핑합니다.
    *   (예: 속성 공유 규칙) 서로 다른 피의자 갑과 을이 한 번이라도 같은 중국 IP(`vt_ip`)를 거쳐 접속했다면? ➡️ **[합숙/공범 가능성 90%]** 관계를 즉시 도출해 냅니다.

## 🚀 4. 기대 효과 및 결론
CCOP 온톨로지 모델은 경찰/금융권(KICS 등)의 파편화된 데이터를 **가장 인간(수사관)의 직관에 가까운 형태로 재조립**해 줍니다. 
이 4-Layer 기반의 지식 그래프는 **복잡한 자금 세탁의 은닉처를 단숨에 가시화**하고, 시맨틱 기반의 자동 추론 엔진이 **미처 발견하지 못한 점조직의 배후까지 인텔리전스로 엮어내는** 현대 사이버 범죄 수사의 가장 강력한 코어 엔진이 될 것입니다.
