# i2 Analyst's Notebook vs CCOP 시스템 비교 분석

**작성일**: 2026-02-03  
**버전**: 1.0  
**목적**: IBM i2 Analyst's Notebook과 CCOP 시스템의 RDB/GDB 활용 방식 비교 분석

---

## 1. 개요

### 1.1 i2 Analyst's Notebook
IBM에서 개발한 범죄 및 정보 분석 전문 도구로, 수사관이 복잡한 데이터를 시각적으로 분석하고 관계를 파악할 수 있도록 지원합니다.

### 1.2 CCOP 시스템
KICS 온톨로지 기반의 사이버 범죄 분석 통합 플랫폼으로, AI 자동화 기능과 웹 기반 협업을 특징으로 합니다.

## 2. 아키텍처 비교

| 구분 | i2 Analyst's Notebook | CCOP 시스템 |
|------|----------------------|-------------|
| **데이터베이스** | iBase (PostgreSQL 기반 전용 DB) | PostgreSQL + AgensGraph (통합 DB) |
| **그래프 엔진** | 자체 엔진 (Entity-Link 모델) | AgensGraph (Cypher 기반) |
| **쿼리 언어** | GUI 기반 필터링 | Cypher + 자연어 (Text-to-Cypher) |
| **배포 방식** | 데스크톱 앱 (Windows 전용) | 웹 기반 (브라우저) |
| **라이선스** | 상용 (고가) | 오픈소스 기반 |

## 3. 데이터 처리 흐름 비교

### 3.1 i2 데이터 처리 흐름

```
CSV/Excel 파일
      ↓
[수동 매핑] ← 분석관이 직접 Entity Type, Link Type 정의
      ↓
iBase 저장 (RDB)
      ↓
[수동 분석] ← 분석관이 직접 관계 연결
      ↓
Entity-Link 차트 시각화
```

**특징:**
- 수작업 중심의 데이터 매핑
- 분석관의 도메인 지식에 의존
- 정확하지만 시간 소요 큼

### 3.2 CCOP 데이터 처리 흐름

```
CSV 파일 업로드
      ↓
[AI 자동 매핑] ← LLM이 KICS 온톨로지 기반 매핑 추천
      ↓
RDB 저장 (정규화)
      ↓
[자동 변환] ← RdbToGraphService
      ↓
GDB 저장 (관계 최적화)
      ↓
[AI 분석] ← Text-to-Cypher, 패턴 인식
      ↓
그래프 시각화
```

**특징:**
- AI 자동화로 시간 단축
- KICS 표준 일관성 보장
- 대용량 데이터 처리 가능

## 5. RDB/GDB 활용 방식 비교

### 5.1 i2의 데이터베이스 활용

```
┌─────────────────────────────────────────────────────────────┐
│                    i2 데이터 모델                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  iBase (RDB 기반)                                            │
│  ├── Entity 테이블: 사람, 계좌, 전화번호 등                  │
│  ├── Link 테이블: 관계 정보                                  │
│  └── Attribute 테이블: 속성 정보                             │
│                                                              │
│  특징:                                                       │
│  - RDB만 사용 (별도 GDB 없음)                                │
│  - Entity-Link 모델로 관계 표현                              │
│  - JOIN 연산으로 관계 조회                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 CCOP의 데이터베이스 활용

```
┌─────────────────────────────────────────────────────────────┐
│                    CCOP 데이터 모델                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  PostgreSQL (하이브리드 DB)                                  │
│  ├── RDB 영역                                                │
│  │   ├── rdb_cases (사건)                                    │
│  │   ├── rdb_accounts (계좌)                                 │
│  │   ├── rdb_phones (전화번호)                               │
│  │   ├── rdb_transfers (이체 내역)                           │
│  │   ├── rdb_calls (통화 내역)                               │
│  │   └── rdb_relations (관계)                                │
│  │                                                           │
│  └── GDB 영역 (AgensGraph)                                   │
│      ├── vt_case (사건 노드)                                 │
│      ├── vt_bacnt (계좌 노드)                                │
│      ├── vt_telno (전화번호 노드)                            │
│      ├── vt_psn (인물 노드)                                  │
│      └── digital_trace (관계 엣지)                           │
│                                                              │
│  특징:                                                       │
│  - RDB: 원본 데이터 저장, 정규화, CRUD                       │
│  - GDB: 관계 탐색 최적화, Cypher 쿼리                        │
│  - RDB → GDB 자동 변환 지원                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 쿼리 방식 비교

**i2 쿼리 (GUI 기반)**
```
1. Entity 선택 → "사람" 타입 필터
2. 속성 조건 추가 → 이름 = "홍길동"
3. 연결된 Entity 조회 → GUI 클릭
```

**CCOP 쿼리 (Cypher)**
```cypher
-- 직접 Cypher 쿼리
MATCH (p:vt_psn {name: '홍길동'})-[r]->(other)
RETURN p, r, other

-- 또는 자연어 쿼리
"홍길동과 연결된 모든 계좌를 찾아줘"
→ AI가 자동으로 Cypher 변환
```

## 8. 연동 및 상호 보완 전략

### 8.1 추천 하이브리드 워크플로우

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    최적의 분석 워크플로우                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  [1단계: CCOP - 데이터 전처리]                                          │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │  CSV 업로드 → AI 자동 매핑 → RDB 저장 → GDB 변환                 │  │
│  │  - KICS 온톨로지 자동 적용                                        │  │
│  │  - 관계 자동 추론                                                 │  │
│  │  - 노드 자동 병합                                                 │  │
│  │  - Text-to-Cypher 초기 분석                                       │  │
│  └──────────────────────────────────────────────────────────────────┘  │
│                              ↓                                          │
│                        [데이터 내보내기]                                │
│                              ↓                                          │
│  [2단계: i2 - 고급 시각화]                                              │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │  CCOP 데이터 임포트 → Entity-Link 차트                            │  │
│  │  - 타임라인 분석                                                  │  │
│  │  - 지도 분석                                                      │  │
│  │  - 고급 네트워크 시각화                                           │  │
│  │  - 보고서 생성                                                    │  │
│  └──────────────────────────────────────────────────────────────────┘  │
│                                                                          │
│  [효과]                                                                 │
│  - CCOP의 자동화로 데이터 준비 시간 80% 단축                           │
│  - i2의 시각화로 프레젠테이션 품질 향상                                │
│  - 두 시스템의 장점 극대화                                             │
└─────────────────────────────────────────────────────────────────────────┘
```

### 8.2 i2 호환 데이터 내보내기 API

CCOP에서 i2로 데이터를 내보내기 위한 API 엔드포인트:

```
GET /api/v1/export/i2?graph={graph_name}&format={csv|anb}
```

**지원 형식:**
- CSV: Entity, Link, Attribute 테이블 분리
- ANB: i2 네이티브 형식 (향후 지원)

3. **보고서 자동 생성** 기능
4. **오프라인 모드** 지원
