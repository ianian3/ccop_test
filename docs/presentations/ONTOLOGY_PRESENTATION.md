# 1️⃣ 온톨로지란?

## 정의

> **"특정 도메인의 개념과 그 관계를 명시적으로 정의한 지식 체계"**

```
일반 데이터베이스              vs              온톨로지
─────────────────                        ─────────────
테이블, 컬럼                               개념(Entity), 관계(Relationship)
데이터 저장                                의미(Semantics) 표현
SQL 조회                                   추론(Inference) 가능
```

## 왜 수사에 필요한가?

| 기존 방식 | 온톨로지 방식 |
|----------|-------------|
| "계좌번호 110-234-567890 검색" | "이 계좌와 연결된 모든 사람과 사건" |
| 단순 키워드 검색 | **관계 기반 탐색** |
| 수동 연결 분석 | **자동 패턴 탐지** |

# 2️⃣ CCOP 온톨로지 개요

## 핵심 특징

| 특징 | 설명 |
|-----|------|
| 🏛️ **KICS 표준** | 한국 경찰청 수사정보시스템 준수 |
| ⚖️ **법적 분류** | 모든 엔티티에 법적 카테고리 명시 |
| 🔍 **패턴 탐지** | 5가지 범죄 유형 자동 인식 |
| 🤖 **LLM 추론** | GPT 기반 관계 자동 추론 |

## 3가지 핵심 구성요소

```
┌─────────────────────────────────────────────────┐
│            CCOP 온톨로지 시스템                   │
├─────────────────────────────────────────────────┤
│                                                 │
│   ┌───────────┐  ┌───────────┐  ┌───────────┐  │
│   │ ENTITIES  │  │ RELATIONS │  │ INFERENCE │  │
│   │  엔티티    │  │   관계     │  │    추론    │  │
│   │  (What)   │  │  (How)    │  │   (Why)   │  │

│                                                 │

# 3️⃣ 엔티티 설계

## 계층 구조

```
                         Entity
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
     ┌──────┐          ┌───────┐         ┌────────┐
     │ Case │          │Suspect│         │Evidence│
     │ 사건  │          │ 용의자 │         │  증거   │
     └──────┘          └───┬───┘         └───┬────┘
                           │                 │
                    ┌──────┼──────┐    ┌─────┼─────┐
                    ▼      ▼      ▼    ▼     ▼     ▼
                  연락처  디지털ID     금융  디지털  물리
                  전화번호  SNS계정    계좌   IP    ATM
```

## 주요 엔티티 (9종)

| 카테고리 | 엔티티 | 노드 라벨 | 법적 분류 |
|---------|-------|----------|----------|
| **사건** | Case | `vt_flnm` | 수사사건 |
| **용의자** | Person | `vt_psn` | 피의자정보 |
| | Phone | `vt_telno` | 통신사실확인자료 |
| **금융** | Account | `vt_bacnt` | 금융거래정보 |
| **디지털** | IP | `vt_ip` | 통신자료 |
| | Site | `vt_site` | 인터넷기록 |
| | File | `vt_file` | 디지털증거 |
| **물리** | ATM | `vt_atm` | 물리증거 |

# 4️⃣ 관계 설계

## Case 중심 Star 구조

```
                    ┌──────────────┐
                    │     사건      │
                    │   (Case)     │
                    └──────┬───────┘
                           │
      ┌──────────┬─────────┼─────────┬──────────┐
      ▼          ▼         ▼         ▼          ▼
  used_phone  used_account  digital_trace  accessed_ip  visited_site
      │          │         │         │          │
      ▼          ▼         ▼         ▼          ▼
   전화번호      계좌       파일       IP        사이트
```

## 관계별 의미

| 관계 | 의미 | 법적 분류 |
|-----|------|----------|
| `used_phone` | "사건에서 이 전화를 사용함" | 통신사실확인자료 |
| `used_account` | "사건에서 이 계좌를 사용함" | 금융거래정보 |
| `digital_trace` | "사건의 디지털 흔적" | 디지털증거 |
| `accessed_ip` | "이 IP에서 접속함" | 통신자료 |
| `visited_site` | "이 사이트를 방문함" | 인터넷기록 |

```
입력 (CSV 데이터)                   출력 (Enriched 노드)
─────────────────                   ─────────────────────
{                                   {
  "actno": "110-234-567890"           "actno": "110-234-567890",
}                                     "ontology_type": "FinancialEvidence",
                                      "entity_subtype": "BankAccount",
         ──────────────►              "legal_category": "금융거래정보",
         OntologyEnricher             "kics_compliant": true
                                    }
```

## 추론 규칙

```
┌─────────────────────────────────────────────────────┐
│  규칙 1: 조직범죄 탐지                                │
│  ─────────────────────                               │
│  IF   동일 전화/계좌가 3건 이상 사건에서 사용           │
│  THEN 조직범죄 가능성 (신뢰도 80%)                    │
│  법적 근거: 범죄수익은닉규제법                         │
├─────────────────────────────────────────────────────┤
│  규칙 2: 자금세탁 탐지                                │
│  ─────────────────────                               │
│  IF   3단계 이상 계좌이체 경로 존재                    │
│  THEN 자금세탁 의심 (신뢰도 75%)                      │
│  법적 근거: 특정금융거래정보법                         │
└─────────────────────────────────────────────────────┘
```

# 6️⃣ 활용 사례

## 6.1 범죄 패턴 탐지

```
질문: "몸캠피싱 패턴 사건 찾기"

Cypher 쿼리:
MATCH (c:vt_flnm)-[:digital_trace]->(s:vt_site)
MATCH (c)-[:related_to]->(f:vt_file)
MATCH (c)-[:used_account]->(a:vt_bacnt)
WHERE s.site CONTAINS 'chat'
RETURN c, s, f, a

결과: 3건 탐지
```

## 6.2 대포통장 탐지

```
질문: "여러 사건에서 사용된 의심 계좌"

Cypher 쿼리:
MATCH (c:vt_flnm)-[:used_account]->(a:vt_bacnt)
WITH a, count(DISTINCT c) AS cnt
WHERE cnt >= 3
RETURN a.actno, cnt

결과:
┌──────────────────┬─────┐
│ 계좌번호          │ 연루 │
├──────────────────┼─────┤
│ 110-234-567890   │  5  │  ← 대포통장 의심
│ 223-456-789012   │  3  │
└──────────────────┴─────┘
```

# 7️⃣ 향후 계획

## 현재 vs 목표 아키텍처

| 항목 | 현재 (v1.x) | 목표 (v2.x) |
|-----|-------------|-------------|
| 그래프 구조 | Heavy Graph | **Lightweight Graph** |
| 원천 저장소 | GDB | **RDB (SSOT)** |
| 노드 속성 | 모든 속성 저장 | **rdb_id 참조만** |
| 온톨로지 메타 | 노드에 직접 | **RDB 테이블 분리** |

## 개선 로드맵

```
v1.2 ─────────────────────────────────────────► v2.0
 │                                               │
 ├─ 역관계(inverse) 정의 추가                      │
 ├─ Person/Organization 엔티티 추가               │
 │                                               │
 │                            ┌─────────────────┘
 │                            │
 │                            ├─ RDB 스키마 설계
 │                            ├─ Lightweight Graph 전환
 │                            └─ GraphRAG 하이브리드 완성
```

# 🎯 요약

## 3줄 요약

```
1. CCOP 온톨로지는 KICS 표준 기반 사이버 범죄 지식 그래프입니다.
2. 9종 엔티티 + 6종 관계 + 2종 추론 규칙으로 구성됩니다.

| 🔗 | **연결**: 흩어진 수사 데이터를 하나의 그래프로 |
